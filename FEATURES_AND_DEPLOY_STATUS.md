# Movie Swipe — 実装機能一覧とデプロイ状況

## 1. ローカルで実装されている機能（一覧）

### コア体験
| 機能 | 説明 |
|------|------|
| 映画スワイプ | 右＝見たい、左＝スキップ、下＝見たことある、上＝お気に入り |
| 二つの山システム | 20本スワイプで `/choose` に遷移し、見たい山から1本を決定 |
| 選択タイム | ランダム選択（ルーレット）・手動選択 → 「今すぐ見る」「後で見る」 |
| 映画キュー（プリフェッチ） | 次の数本を先読みしてスワイプ後の表示を高速化（ローカル側でキュー数・閾値の違いあり） |

### フィルター・レコメンド
| 機能 | 説明 |
|------|------|
| フィルター | ジャンル・上映時間・リリース年・配信サービスで絞り込み |
| シンプル/詳細モード | 気分プリセット（「疲れてる」「暇してる」「友達と見る」）と詳細条件の切り替え |
| 時間帯レコメンド | 現在時刻に応じたフィルター提案（22時以降＝90分以内、平日夜＝2時間以内など） |

### 一覧・統計・共有
| 機能 | 説明 |
|------|------|
| 選んだリスト（`/selected`） | 「今すぐ見る」「後で見る」の一覧、配信リンク、個別シェア・削除・見たマーク |
| まとめてシェア | 選択モードで複数本を選び、まとめて Web Share API / クリップボードで共有 |
| お気に入り（`/favorites`） | 上スワイプで登録、一覧表示・削除、統計との連携 |
| 統計（`/stats`） | スワイプ総数・選んだ数・割合・今すぐ/後で見る数・お気に入り数、今日/今週/今月/全期間、プログレスバー表示 |

### 複数人・投票
| 機能 | 説明 |
|------|------|
| 投票モード（`/vote`, `/vote/[sessionId]`） | セッション作成・参加、同じ10本に「いいね」投票、集計で1位表示、2秒ごと更新 |

### 映画データ・表示
| 機能 | 説明 |
|------|------|
| 日本語優先表示 | TMDb の `language=ja-JP` と日本語詳細取得でタイトル・あらすじを日本語優先、不足時は英語フォールバック |
| カード詳細モーダル | タップで監督・出演者（上位5人）・配信サービス・上映時間・リリース年・あらすじを表示 |
| 「今すぐ見る」 | 配信サービス（Netflix 等）の作品ページへ遷移（`providerUrls` 等でリンク生成） |
| 同じ映画を出さない | `shownMovies` / `watchedMovies` を localStorage で管理し、表示済み・視聴済みを除外 |
| 404対策 | 条件に合う映画がなくなったらメッセージ表示のうえ、自動でフィルターリセット |

### UI/UX
| 機能 | 説明 |
|------|------|
| スマホ最適化 | 縦向き固定・ズーム無効、`touchAction` でスワイプ時のスクロール抑制、アニメ短縮 |
| ヘッダー整理 | 常時は「🎬 フィルター」「選んだリスト」、その他は「⋯ メニュー」に集約 |

### データ・ユーティリティ（ローカル専用ディレクトリに存在）
| 機能 | 説明 |
|------|------|
| データエクスポート/インポート | 選んだリスト・統計・表示済み等を JSON でエクスポート/インポート（`DataBackupModal` + `dataExport`） |
| ストレージ使用量表示 | エクスポート画面で localStorage の使用量を表示 |
| デバイス判定 | `deviceDetection`（モバイル判定など） |
| エラーハンドリング | `errorHandler`（パース等） |
| 配信URL取得 | `providerUrls`（「今すぐ見る」リンク生成） |
| API レート制限対策 | `apiRateLimiter`（将来用にコードのみ存在、API 側では未使用の記載あり） |

## 2. デプロイできていない機能

以下は **ローカル開発用ディレクトリ**（`movie-swipe/movie-swipe/movie-swipe/`）にのみ実装があり、**デプロイ用ディレクトリ**（`movie-swipe/movie-swipe/`）には含まれていない（またはファイルが存在しない）ため、**現時点ではデプロイされていません**。

| 機能 | 内容 |
|------|------|
| **データエクスポート/インポート** | `DataBackupModal` および `app/utils/dataExport.ts` がデプロイ用の `app` には存在しない。統計・選んだリスト・お気に入り・表示済み等を JSON でバックアップ/復元する機能は本番では利用不可。 |
| **データバックアップ用 UI** | 統計ページ・選んだリスト・お気に入りページから開く「データのエクスポート/インポート」モーダルは、デプロイ用には `DataBackupModal` が無いため表示されない。 |
| **app/utils 一式** | デプロイ用の `app` 配下に `utils` フォルダが無く、以下のモジュールは本番に存在しない。<br>・`dataExport.ts`（エクスポート/インポート）<br>・`providerUrls.ts`（配信サービス視聴URL）<br>・`deviceDetection.ts`（モバイル判定）<br>・`errorHandler.ts`（エラー処理）<br>・`apiRateLimiter.ts`（レート制限対策）<br>※ デプロイ用の `app/choose/page.tsx` 等が `../utils/providerUrls` を参照している場合、デプロイ先ではビルドエラーまたは実行時エラーになる可能性がある。 |
| **映画キューの閾値・挙動の差異** | ローカル側では「残り本数が 2〜3 本で補充」などキュー補充の閾値が異なる実装になっている可能性がある。デプロイ用とローカルで同一挙動かは要確認。 |

## 3. デプロイ状況の整理

- **デプロイ先**: Vercel（例: https://movie-swipe-delta.vercel.app/ ）
- **デプロイ用ディレクトリ**: `movie-swipe/movie-swipe/`（リポジトリルートや Vercel の Root Directory の設定に依存）
- **ローカル開発用ディレクトリ**: `movie-swipe/movie-swipe/movie-swipe/`（上記「実装されている機能」の多くはここで開発・動作確認されている）

デプロイ用に「ローカルで実装済みだが未反映」の機能を反映するには、少なくとも次が必要です。

1. **DataBackupModal** と **app/utils/** 以下（`dataExport.ts`, `providerUrls.ts`, `deviceDetection.ts`, `errorHandler.ts`, `apiRateLimiter.ts`）をデプロイ用の `app` にコピーまたはマージする。
2. 統計・選んだリスト・お気に入りページで `DataBackupModal` を開く UI がデプロイ用にも入っているか確認し、不足していれば追加する。
3. デプロイ用の `app/choose/page.tsx` 等が `utils` を参照している場合、上記 1 で `utils` を追加すれば「今すぐ見る」リンク等が本番でも動作する。
