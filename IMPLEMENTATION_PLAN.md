## 1. 高優先度（REVIEW ベース）

### 1-1. レート制限対策
- **状態**: 将来用としてコードのみ残存（`app/utils/apiRateLimiter.ts`）
- **理由**: 現状は通常の fetch で運用。ユーザー数が増えたとき用
- **再実装の目安**: ユーザー100人以上 or 429エラーが頻発したとき

### 1-2. データエクスポート/インポート ✅
- **状態**: 完了
- **内容**: 選んだ映画・お気に入り・統計を JSON でエクスポート/インポート、使用量表示
- **配置**: `/stats`・`/favorites`・`/selected` に「データのバックアップ」リンク

### 1-3. エラーハンドリング強化（Phase 3）
- **状態**: 未実装
- **内容**:
  - エラータイプの分類（ネットワーク / API / データ）
  - ユーザー向けメッセージの整理
  - オフライン検知とメッセージ表示
- **想定ファイル**: `app/utils/errorHandler.ts`、`app/page.tsx`、API ルートのエラーレスポンス

---

## 2. ユーザー要望（「今見る」で配信サービスへ）

### 2-1. 「今見る」から配信サービスへ直接遷移
- **状態**: 未実装
- **要望**: 今見るを押したらそのまま配信サービスに移る。スマホはアプリ、PCはサイト
- **内容**:
  - provider_id（Netflix / Prime / Disney+ 等）ごとに URL を生成
  - モバイル判定 → アプリスキーム or Web URL
  - 対象: メイン画面モーダル、選択タイム（/choose）、選んだ映画（/selected）
- **想定**: `app/utils/providerUrls.ts`、`app/utils/deviceDetection.ts`、各ページの「今見る」リンク修正

### 2-2. ロゴタップでホームに戻る
- **状態**: ✅ 完了（全ページで Link で実装済み）

## 3. 中優先度（REVIEW ベース）

### 3-1. 画像最適化
- Next.js `<Image>` への移行
- レスポンシブ（srcset）、遅延読み込み、WebP 検討

### 3-2. キャッシング戦略
- Next.js キャッシュの活用

### 3-3. オンボーディング
- 初回訪問時の簡単なチュートリアル
- スワイプ操作の説明（左/右/上/下）
